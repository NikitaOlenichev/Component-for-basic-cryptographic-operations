# Компонент для базовых криптографических операций.
Данный компонент предоставляет набор классов на C++ для выполнения основных криптографических задач: вычисление хеш-сумм, создание и проверка усиленной неквалифицированной электронной подписи (УНЭП), а также симметричное и асимметричное шифрование/дешифрование данных.
# Структура репозитория
В репозитории 5 веток:
1) Main - основная ветка, в которую в конце будут влиты все остальные ветки
2) Decription - ветка для функции дешифрования
3) Encription - ветка для функции шифрования
4) Hash - ветка для функции подсчета хэша
5) Signatures - ветка для функции генерации подписей
# Основные возможности
- **Хеширование**: Вычисление контрольных сумм и хеш-значений данных.
- **Шифрование и Дешифрование**:
  - **Симметричные алгоритмы** (AES, ...).
  - **Асимметричные алгоритмы** (RSA, ...).
- **Электронная подпись**: Генерация и проверка усиленной неквалифицированной электронной подписи (УНЭП).
# Структура компонента
Компонент состоит из 4 классов:
1) **Класс для подсчета хеша**  
   Класс, предназначенный для вычисления хешей различных данных, что является основой для проверки целостности данных.
2) **Класс для шифрования**  
   Класс предоставляет методы для шифрования данных с использованием симметричных и асимметричных алгоритмов:
   - **Симметричное шифрование**: алгоритм, использующий для шифрования и дешифрования один и тот же ключ.
   - **Асимметричное шифрование**: алгоритм, использующий пару ключей (открытый и закрытый).
3) **Класс для дешифрования**  
   Класс для дешифрования данных, зашифрованных с использованием симметричных и асимметричных алгоритмов:
   - **Симметричное дешифрование**: алгоритм, использующий для шифрования и дешифрования один и тот же ключ.
   - **Асимметричное дешифрование**: алгоритм, использующий пару ключей (открытый и закрытый).
4) **Класс для генерации подписей УНЭП**  
   Класс для работы с криптографической подписью, используемой в системе электронного документооборота.
Все классы спроектированы с учётом принципов единственной ответственности и могут использоваться как независимо, так и в составе более крупных систем.
